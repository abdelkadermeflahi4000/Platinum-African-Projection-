# 🤝 دليل المساهمة | Contributing Guide

شكراً لاهتمامك بالمساهمة في مشروع **Beyond the Facade**! هذا الدليل سيساعدك على البدء.

---

## 🌟 فلسفة المساهمة

نحن نؤمن بأن التنوع في الخلفيات (تقنية، فلسفية، فنية) يثري المشروع. لا توجد مساهمة "صغيرة" جداً أو "كبيرة" جداً.

### أنواع المساهمات المقبولة:
- 💻 **الكود**: ميزات جديدة، إصلاح أخطاء، تحسين أداء
- 📝 **الوثائق**: ترجمة، توضيحات، أمثلة
- 🎨 **الفن الرقمي**: تصاميم بصرية، واجهات، تجارب غامرة
- 🧠 **الفلسفة**: أطر نظرية، تحليلات، نقاشات عميقة
- 🐛 **التقارير**: إبلاغ عن مشاكل، اقتراح تحسينات
- 📢 **النشر**: مشاركة المشروع، ورش عمل، محاضرات

## 🚀 البدء السريع للمطورين

### 1. إعداد البيئة

```bash
# استنساخ المشروع
git clone https://github.com/yourusername/beyond_the_facade.git
cd beyond_the_facade

# إنشاء بيئة افتراضية
python -m venv venv
source venv/bin/activate  # على Windows: venv\Scripts\activate

# تثبيت التبعيات
pip install -r requirements.txt

# تشغيل الاختبارات
pytest tests/
```

### 2. اختيار مهمة

تصفح [قائمة المهام](docs/ROADMAP.md) أو ابحث عن issues المفتوحة:
- 🔰 `good first issue`: مثالية للمبتدئين
- 🐛 `bug`: إصلاح مشاكل
- ✨ `enhancement`: ميزات جديدة
- 📚 `documentation`: تحسين الوثائق

### 3. كتابة الكود

```python
# مثال: إضافة طبقة واقع مخفية جديدة
from src.reality_engine import HiddenLayer

new_layer = HiddenLayer(
    id="layer_4",
    name="الطبقة الكمومية",
    description="طبقة التراكب الكمومي حيث تتواجد الاحتمالات المتعددة",
    unlock_condition="quantum_observation"
)

# أضف اختبارات للطبقة الجديدة
def test_quantum_layer_unlock():
    reality = RealityEngine()
    reality.expand_awareness(mode="omnidirectional")
    reality.awareness_level = 1.0
    assert reality._check_unlock_condition(new_layer) == True
```

### 4. معايير الكود

- ✅ اتبع نمط PEP 8 لبايثون
- ✅ اكتب docstrings واضحة (عربي + إنجليزي)
- ✅ أضف اختبارات لكل ميزة جديدة
- ✅ تأكد من عدم كسر الاختبارات الحالية
- ✅ استخدم أسماء ذات معنى (عربية أو إنجليزية)

### 5. إرسال Pull Request

```bash
# أنشئ فرعاً جديداً
git checkout -b feature/my-new-layer

# أضف تغييراتك
git add src/reality_engine.py tests/test_layers.py

# التزم برسالة واضحة
git commit -m "feat: إضافة الطبقة الكمومية مع اختبارات"

# ادفع الفرع
git push origin feature/my-new-layer

# أنشئ PR على GitHub
```

## 📝 إرشادات الكتابة

### للوثائق:
- استخدم لغة واضحة وبسيطة
- قدم أمثلة عملية
- ترجم المحتوى (عربي + إنجليزي)
- استخدم Markdown بشكل صحيح

### للكود:
```python
# ❌ سيء
def f(x): return x*2

# ✅ جيد
def expand_bounds(factor: float) -> BoundingBox:
    """
    توسيع حدود الواقع بعامل معين

    Args:
        factor: عامل التوسع (مثلاً 2.0 لتضاعف الحجم)

    Returns:
        BoundingBox: النظام الموسع الجديد
    """
    return bounding_box.expand(factor)
```

### للرسائل (Commits):
```
feat: إضافة نظام المنظور الشامل
fix: إصلاح خطأ في حساب الحدود
docs: تحديث دليل المساهمة
test: إضافة اختبارات للحلقة الزمنية
refactor: إعادة هيكلة محرك الواقع
```

## 🎨 المساهمة الفنية

إذا كنت فناناً رقمياً أو مصمماً:

### ما يمكنك تقديمه:
- تصاميم للطبقات المخفية
- واجهات مستخدم ثلاثية الأبعاد
- تجارب سمعية-بصرية
- هوية بصرية للمشروع

### الأدوات المقترحة:
- Blender: للنمذجة ثلاثية الأبعاد
- Three.js: للعرض في المتصفح
- ShaderToy: للتأثيرات البصرية

### كيفية الإرسال:
1. احفظ ملفاتك في `assets/`
2. أضف README يشرح التصميم
3. أرسل PR مع صور توضيحية

## ❓ الأسئلة الشائعة

### س: هل يجب أن أكون خبيراً؟
ج: كلا! نرحب بجميع المستويات. ابدأ بـ `good first issue`.

ج: تصفح القضايا المفتوحة، اختر ما يناسب مهاراتك واهتماماتك.

ج: عادةً خلال أسبوع. سنقدم ملاحظات بناءة.
